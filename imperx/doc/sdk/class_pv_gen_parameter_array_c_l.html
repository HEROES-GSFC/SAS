void PvGenParameterArrayCL::GetDeviceTemplates ( PvStringList &  aDeviceTemplateList  )  [static]

Static method returning the list of all available device templates.

A device template is used as a hint when probing cameras on a specific port. The device template list is built from all available CLProtocol.dll on the system. One DLL can support more than one device template. The strings returned are properly formatted and ready to be used in a GUI.

PvString PvGenParameterArrayCL::ProbeDevice ( const PvString &  aPortID,
const PvString &  aDeviceTemplate,
PvUInt32  aSerialTimeout = 500 
) [static]

Using a device template, a specific port is probed for a camera.

If a camera is found a camera ID is returned to the caller. That camera ID can be used to connect to camera.

Parameters:
[in] aPortID Port to use when probing the device. Available ports are enumerated with GetPortIDs. With the eBUS SDK the enumerated ports are highly likely going to be named PvSerialBridge used in Camera Link DLL mode.
[in] aDeviceTemplate Allows selecting which CLProtocol library to use and can even refinge the camera selection within the library. All possible device templates for the system can be enumerated with GetDeviceTemplates.
[in] aSerialTimeout Optional serial timeout used when probing the camera. 500 ms isued as a default value if not provided.
Returns:
Camera ID if the device is found. An empty string otherwise.
