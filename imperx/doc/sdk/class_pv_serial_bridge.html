Detailed Description

Bridges a COM port of Camera Link DLL to an IP Engine serial port.

Note:
This class is only available on Windows.

The serial bridge takes all the data coming from either a COM port or eBUS SDK Camera Link DLL (clserpte.dll) and sends to an IP Engine serial port.

It also takes all the data coming from an IP Engine serial port and sends it to either a COM port or eBUS SDK Camera Link DLL.

Depending on the Start method that is used, the bridge is either started as a COM port bridge or Camera Link DLL bridge.

Statistics can be retrieved about a bridge: count of transmitted and received bytes.

The COM port can be used with a stand-alone application, usually a camera configuration application relying on a COM port (ie: COM1, COM2, etc.) to interact with the camera. The port selected by the application must be connected with a NULL modem to the port used by the bridge. It can be two real COM ports connected together or two virtual ports with a COM port emulation software/driver.

The Camera Link DLL can be used with a stand-alone application, usually a camera configuration application, relying on a the API described in the Camera Link standard used to interface a serial port. Instead of being the serial port of a frame grabber here it is the serial port of a GigE Vision IP Engine.

The Camera Link DLL can also be used with a PvGenParameterArrayCL - a GenICam CLProtocol interface.

For a simple example illustrating how to use PvSerialBridge, refer to the sample entitled PvSerialBridgeSample.

PvResult PvSerialBridge::SetHardCodedBaudRate ( PvUInt32  aBaudRate  ) 

Sets the IP Engine hard-coded baud rate (user-defined).

If the IP Engine GenICam interface does not have the features to control the configuration of the serial port used by the bridge, the developer must know and provide the baud rate to which the serial port is hard-coded.

This hard-coded baud rate is only used to control which unique baud rate is made available to the application or CLProtocol DLL using the Camera Link DLL. This unique baud rate must match the hard-coded IP Engine serial port baud rate.

This method should be called after starting the Camera Link DLL bridge, but before starting the application using the Camera Link DLL or before creating the CLProtocol node map.

With a serial COM port, the baud rate of the COM port is specified through the PvSerialPortConfiguration parameter passed to the Start method and simply isn't used to set the IP Engine serial port baud rate. The hard coded baud rate does not need to be defined.

If the IP Engine GenICam interface does not have the features to control the configuration of the serial port and this method is not called to specify the hard-coded baud rate of the serial port, 9600 baud is used/assumed.

Valid baud rates are the intersection between the IP Engine serial port baud rates and what is described in the Camera Link specification:

  • 9600 baud
  • 19200 baud
  • 38400 baud
  • 57600 baud
  • 115200 baud

PvSerialPortConfiguration  aSerialPortConfiguration,
PvDevice *  aDevice,
PvIPEngineSerial  aDevicePort

Starts the serial bridge using a serial port as the source.

Supported baud rates are 9600, 14400, 19200, 28800, 38400, 57600, 115200. For parity, none/even/odd are supported and expressed as N/E/O. Supported bit counts are 7 and 8. Supported stop bits are 1 and 2.

Parameters:
[in] aSerialPort String identifying which serial port on the host to use. Example: "COM1".
[in] aSerialPortConfiguration An object containing the serial port configuration to be applied on aSerialPort.
[in] aDevice Pointer to a connected PvDevice used to open the device-side serial port.
[in] aDevicePort Device port to bridge to. Example: PvIPEngineBulk0.
Returns:
Includes:
  • PvResult::Code::OK
  • PvResult::Code::INVALID_PARAMETER if a parameter is not properly formatted. Use PvResult::GetDescription to get more information.
  • PvResult::Code::NOT_CONNECTED if the device is not connected.
  • PvResult::Code::NOT_SUPPORTED if the aDevicePort is not supported by the device.
