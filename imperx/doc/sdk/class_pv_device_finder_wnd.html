Detailed Description

GUI-based IP Engine finder.

PvDeviceFinderWnd is a GUI-based object that finds all IP Engines on a network and lets the user select one.

To find an IP Engine (GUI):

  1. Create a GUI-based IP Engine finder (PvDeviceFinderWnd object).
  2. Set the window's properties. Use SetPosition and other PvWnd Set* methods.
  3. Display the window. Use ShowModalor ShowModeless.
  4. Retrieve the user's selection (information about the chosen IP Engine). Use PvDeviceFinderWnd::GetSelected.

To connect to the IP Engine, see PvDevice.

bool PvDeviceFinderWnd::OnFound ( PvDeviceInfo *  aDI  )  [virtual]

Define the behavior of a user prompt (PvDeviceFinderWnd object) when it finds an IP Engine.

When the PvDeviceFinderWnd object finds an IP Engine, it automatically calls the OnFound method. If OnFound returns true, the PvDeviceFinderWnd object adds the IP Engine to the list. The default version of OnFound method does nothing and always returns true.

However, you can override the OnFound method so that it does something or returns a value other than true. For example, when PvDeviceFinderWnd finds your desired IP Engine, you can use OnFound to connect to it immediately (using PvDevice::Connect(const PvString&,PvAccessType), without waiting for the search to complete. Similarly, you can use OnFound to filter the list of IP Engines by returning false in some cases (such as if PvDeviceInfo::GetAccessStatus isn't a useful value).

Parameters:
[in] aDI The IP Engine that was found.
Returns:
True. (True if the PvDeviceFinderWnd should include the IP Engine in the list of found IP Engines; false if the IP Engine should be omitted from the list.)
