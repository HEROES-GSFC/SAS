Building on Linux

Building samples or your own applications on Linux

Building samples

  1. At the command line sample, source the set_puregev_env script (source set_puregev_env)
  2. Browse (cd) to the folder of the sample
  3. Simply type make to build the sample
  4. The binary generated is in the same folder as the sample, ready to be run

All samples are using the same master makefile (sample.Makefile) that can be found in the samples folder. It is able to build pure command line samples and those using Qt like GEVPlayer.

Building your own applications

You can look at the sample.Makefile for a makefile that can build applications using the eBUS SDK. You need to source the eBUS SDK environment as described above in the samples section before building against the eBUS SDK.

Important points to consider:

  1. You need to add the include path -I/home/builduser/products/ebus-puregev-package/x86_64/branch_pv_3_0/builds/include
  2. You need to add the library path -L/home/builduser/products/ebus-puregev-package/x86_64/branch_pv_3_0/builds/lib
  3. You need to link against the following libraries (may not all be required)
    1. -lPvBase
    2. -lPvDevice
    3. -lPvBuffer
    4. -lPvGUIUtils
    5. -lPvGUI
    6. -lPvPersistence
    7. -lPvGenICam
    8. -lPvStreamRaw
    9. -lPvStream
    10. -lPvTransmitterRaw
    11. -lPvVirtualDevice

Running your application

You can either source the set_puregev_script (source set_puregev_env) or create your own script to launch your application. For an example, you can look at the GEVPlayer script which launches the GEVPlayer.bin application.

This scripts (and yours) does:

  1. Set the $PUREGEV_ROOT environment variable to the eBUS SDK bin folder
  2. Define the required GENICAM environment variables
  3. Define the location of the GENICAM cache folder
  4. Add the eBUS SDK lib folder to the libraries path
  5. Add the GENICAM libraries folder to the libraries path (Linux64_x64 or Linux32_i86)
